# 🚀 What I Built Today — Full Stack Authentication System

Today, I implemented a complete **Authentication & Authorization system** using the MERN stack, focusing on backend logic with Node.js, Express, MongoDB, and JWT.

---

## 🧠 1. Core Understanding

I learned the difference between:

* **Authentication** → Verifying *who the user is* (Login/Signup)
* **Authorization** → Controlling *what the user can access* (Protected routes)

---

## 🏗️ 2. Backend Setup

* Created a Node.js + Express server
* Connected MongoDB using Mongoose
* Structured project using **MVC architecture**:

  * Models → Database structure
  * Controllers → Business logic
  * Routes → API endpoints

---

## 👤 3. User Model

* Designed a schema with:

  * name
  * email (unique)
  * password

* Learned:

  * Why schema is needed
  * Why email must be unique

---

## 🔐 4. Signup (Registration)

* Created `POST /signup` API
* Implemented:

  * Input validation
  * Duplicate user check
  * Password hashing using **bcrypt**

### Key Concept:

```text
Plain password ❌ → Hashed password ✅
```

---

## 🔑 5. Login System

* Created `POST /login` API
* Implemented:

  * User verification
  * Password comparison using bcrypt
  * JWT token generation

---

## 🎫 6. JWT (JSON Web Token)

Learned how JWT works:

* Token contains:

  * userId (payload)
  * signature (security)
  * expiry time

### Flow:

```text
Login → Generate Token → Send to client → Store → Use in requests
```

---

## 🔐 7. Authorization (Protected Routes)

* Created middleware to:

  * Extract token from headers
  * Verify token using `jwt.verify()`
  * Allow or deny access

### Key Idea:

```text
Valid token → Access granted ✅
Invalid/missing token → Access denied ❌
```

---

## 🌐 8. React Integration

* Built frontend for:

  * Signup
  * Login
  * Welcome page

* Implemented:

  * API calls using fetch
  * Token storage in localStorage
  * Logout functionality

---

## 🔁 9. Full Flow (End-to-End)

```text
Signup → Store user
        ↓
Login → Generate JWT
        ↓
Store token (localStorage)
        ↓
Send token in headers
        ↓
Middleware verifies token
        ↓
Access protected route
```

---

## 🔐 10. Security Improvement (.env)

* Learned to use `.env` for:

  * JWT_SECRET
  * Port

* Replaced hardcoded secret with:

```javascript
process.env.JWT_SECRET
```

## ⚠️ 11. Error Handling

* Added:

  * Input validation
  * Proper status codes (400, 401, 404, 500)
  * Meaningful error messages

## 📊 12. Pagination & Filtering

* Implemented:

  * `page` and `limit`
  * `skip()` and `limit()`

```text
/api/users?page=1&limit=5&search=ayu
```
